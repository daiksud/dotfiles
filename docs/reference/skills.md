# Copilot Skills

カスタムスキルの仕様リファレンスです。

## スキル一覧

| スキル | 説明 |
|--------|------|
| `create-pr` | 変更内容を理解し、適切なコミットメッセージと Description でドラフト PR を作成する |
| `fix-pr` | CI エラーの修正とレビューコメントへの対処を行い、PR をマージ可能な状態にする |

## インストール先

`install.sh` 実行時に `dotfiles/skills/` → `~/.copilot/skills/` へシンボリックリンクされます。

## ディレクトリ構成

```
dotfiles/skills/
├── create-pr/
│   └── SKILL.md
└── fix-pr/
    └── SKILL.md
```

## SKILL.md フォーマット仕様

### 構造

```markdown
---
description: スキルの説明（1行、英語）
name: スキル名
---
# スキル名

## Overview
（スキルが何をするか）

## When to Use
（どのような状況で使うか）

## How to Use
### Step 1: ...
### Step 2: ...

## Output
（スキルの出力内容）

## Constraints
（制約条件）
```

### フロントマター（必須）

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `name` | string | スキルの識別名。ディレクトリ名と一致させる |
| `description` | string | スキルの概要。CLI のスキル一覧に表示される |

### 本文の記述言語

- フロントマター・本文ともに**英語**で記述する
- Copilot CLI がスキルを読み込み、AI エージェントへの指示として使用するため

### セクションガイドライン

| セクション | 必須 | 説明 |
|-----------|------|------|
| Overview | ✅ | スキルの目的を 1〜3 文で説明 |
| When to Use | ✅ | 起動条件やトリガーとなるフレーズ |
| How to Use | ✅ | ステップバイステップの手順 |
| Output | 推奨 | スキル完了時にユーザーに表示する内容 |
| Constraints | 推奨 | 制約事項・失敗時の振る舞い |
| Tips for developers | 任意 | エイリアス設定例など |

## create-pr の詳細仕様

### 動作フロー

1. `git diff` で変更内容を理解
2. Conventional Commits 形式でコミット
3. feature ブランチにプッシュ
4. PR Description を作成しドラフト PR を `gh pr create --draft` で作成
5. 呼び出したユーザーを Assignee に設定

### 入力

- ステージされたファイル、またはコミット済みの差分
- オプション: 変更理由、関連 Issue 番号

### 出力

- 作成されたドラフト PR の URL

## fix-pr の詳細仕様

### 動作フロー

1. 指定された PR の CI ステータスを確認
2. CI 失敗がある場合、ログを分析し修正を繰り返す（最大 3 回）
3. レビューコメントを全件取得し、各コメントの妥当性を判断
4. 妥当なコメントは修正、不当なコメントはスキップ
5. コミット・プッシュ前に `code-review` サブエージェントでローカルレビュー実施
6. プッシュ後、各レビューコメントに対応内容をリプライ

### 入力

- PR 番号（必須）

### 出力

- 各レビューコメントへのリプライ
- CI が通らない場合はエラーレポート
