# Copilot Skills

カスタムスキルの仕様リファレンスです。

## スキル一覧

| スキル      | 説明                                                                              |
| ----------- | --------------------------------------------------------------------------------- |
| `pr-create` | 変更内容を理解し、適切なコミットメッセージと Description でドラフト PR を作成する |
| `pr-fix`    | CI エラーの修正とレビューコメントへの対処を行い、PR をマージ可能な状態にする      |

## インストール先

`install.sh` 実行時に `dotfiles/skills/` → `~/.copilot/skills/` へシンボリックリンクされます。

## ディレクトリ構成

```
dotfiles/skills/
├── pr-create/
│   └── SKILL.md
└── pr-fix/
    └── SKILL.md
```

## SKILL.md フォーマット仕様

### 構造

フロントマターは必須。本文のセクション構成はスキルの性質に応じて自由に定義する。

**シンプルなスキルの例（`pr-create` 形式）:**

```markdown
---
description: スキルの説明（1行、英語）
name: スキル名
---

# スキル名

## 概要

（スキルが何をするか）

## 使用タイミング

（どのような状況で使うか）

## 手順

### ステップ 1: ...

### ステップ 2: ...

## 出力

（スキルの出力内容）
```

**モード分岐があるスキルの例（`pr-fix` 形式）:**

```markdown
---
description: スキルの説明（1行、英語）
name: スキル名
---

# スキル名

## 使い方

（呼び出し方と各モードの説明）

## モード

### mode-a — ...

### mode-b — ...

## 共通ステップ

（全モードで行う処理）

## 制約事項

（制約条件）
```

### フロントマター（必須）

| フィールド    | 型     | 説明                                       |
| ------------- | ------ | ------------------------------------------ |
| `name`        | string | スキルの識別名。ディレクトリ名と一致させる |
| `description` | string | スキルの概要。CLI のスキル一覧に表示される |

### 本文の記述言語

- フロントマターの `description` は**英語**で記述する（CLI のスキル一覧表示に使用されるため）
- 本文は**日本語**で記述する

### セクションガイドライン

| セクション       | 必須 | 説明                                 |
| ---------------- | ---- | ------------------------------------ |
| 概要 or 使い方   | ✅   | スキルの目的や呼び出し方を説明       |
| 手順 or モード   | ✅   | ステップまたはモード別の操作手順     |
| 制約事項         | 推奨 | 制約事項・失敗時の振る舞い           |
| 出力             | 推奨 | スキル完了時にユーザーに表示する内容 |
| 開発者向けヒント | 任意 | エイリアス設定例など                 |

## pr-create の詳細仕様

### 動作フロー

1. `git diff` で変更内容を理解
2. Conventional Commits 形式でコミット
3. feature ブランチにプッシュ
4. PR Description を作成し（本文は日本語で記述）、ドラフト PR を `gh pr create --draft` で作成
5. 呼び出したユーザーを Assignee に設定

### 入力

- ステージされたファイル、またはコミット済みの差分
- オプション: 変更理由、関連 Issue 番号

### 出力

- 作成されたドラフト PR の URL

## pr-fix の詳細仕様

### 動作フロー

1. 指定された PR の CI ステータスを確認
2. CI 失敗がある場合、ログを分析し修正を繰り返す（最大 3 回）
3. レビューコメントを全件取得し、各コメントの妥当性を判断
4. 妥当なコメントは修正、不当なコメントはスキップ
5. コミット・プッシュ前に `code-review` サブエージェントでローカルレビュー実施
6. プッシュ後、各レビューコメントに対応内容をリプライ

### 入力

- PR 番号（必須）

### 出力

- 各レビューコメントへのリプライ
- CI が通らない場合はエラーレポート
